# Get started with Sprite Lab

```text
python -m spritelab v3
```

This starts Sprite Lab. It does not train a model, contact a paid service, or send your images anywhere by itself.

On first launch, choose **Choose image folder**. Then follow the five visible steps:

1. Choose an image folder.
2. Check source and license.
3. Build the dataset.
4. Train.
5. Evaluate and try the model.

Sprite Lab explains anything that needs attention and shows the exact next action. Your original images are not changed.

Training and other potentially costly work show one confirmation immediately before they start. Hosted work also shows the provider, spending limit, and estimated cost.

For a project-local double-click launcher, Sprite Lab can create **Start Sprite Lab.cmd** for Windows and **start-sprite-lab.sh** for macOS or Linux. Existing files with those names are preserved.

Next: [Build a dataset](build_a_dataset.md). For a problem, see [Troubleshooting](troubleshooting.md).
